## 一、项目概述

| 项 | 内容 |
|---|---|
| 项目名 | JDO_automation_portal（车机生态自动化回归平台） |
| 目标 | 车机生态 App 跨品牌发版回归自动化：一次写用例、多品牌跑、自动出报告 |
| 被测对象 | 酷我音乐、喜马拉雅、乐听、爱奇艺、Launcher、车内会议 App 等 |
| 品牌矩阵 | 奥迪、保时捷、大众、宝马、奔驰…… |
| 核心价值 | 替代重复手工回归，量化覆盖率与节省人力 |
| 设计基准 | `doc/原型-交互版.html`（界面/交互以它为准） |

> 8765 端口的 Codex 平台为并行工作区，不合并、不改动，本平台固定 **8770**。

## 二、技术架构

```
┌── 平台层 ────────────────────────────────────────┐
│ 前端  免构建分层静态工程（ES Modules，无打包链）       │
│ 后端  FastAPI + SQLite（api 薄路由 / services 业务）   │
│ 执行  run_service：模拟(节奏=原型) ⇄ 真机 pytest       │
│       多框架适配器（runner_common + media/zcode_runner）│
│ 报告  执行落库 + Allure 真报告（托管 + 预生成秒开）     │
├── 测试框架层（平台可同接多套）────────────────────┤
│ 我们的 framework：pytest+u2+adbutils+POM，case 标记    │
│ Media_automation：ADB-XML dump 驱动，docs CSV 映射     │
│ Media_automation_Zcode：u2 驱动，markdown 覆盖矩阵映射  │
├── 设备层 ────────────────────────────────────────┤
│ Android Automotive 台架（ADB 可用，u2 需 adb root）    │
└──────────────────────────────────────────────────┘
```

## 五、关键决策记录
- **重建（2026-07-03）**：旧实现清空重写。前端选择「免构建分层静态工程」而非 Vue 工程化 ——
  硬性要求与原型交互完全一致（原型为自定义 token 原生实现，组件库必偏离观感），
  且 Python 团队免打包链更好维护；分层由 ES Modules 达成（api/store/ui/theme/router/views）。
- **测试框架技术栈（2026-06-25）**：Appium → **pytest + uiautomator2 + adbutils + POM + Allure**；
  u2 管元素、adbutils 管系统级（am start / dumpsys media_session / 录屏 / logcat）。P0 已实测定稿。
- **覆盖率口径**：业务用例覆盖率（已自动化÷手工总数）+ 预估节省人力；标记自动判定，分子统一收口 `script_case` 表。
  官方库导入后分母含 P1~P4，故看板给「全量 + P1/P2 重点」双口径。
- **平台兼容多框架（2026-07-08）**：平台不强推自己的标记约定，而是每套框架写一个「适配器」把它的
  映射与执行喂进统一的 script_case 表 + 执行内核。当前接三套：
  ① 我们的 framework（`@pytest.mark.case` 标记，u2）；
  ② Media_automation（docs CSV 映射，ADB-XML dump 驱动）；
  ③ Media_automation_Zcode（markdown 覆盖矩阵映射，u2 驱动）。
  共用 `runner_common`（解压/进度插件/实时点亮矩阵），各适配器只写映射解析 + 设备注入差异。
- **框架选型结论（2026-07-08）**：**Zcode（u2 元素级）为长期主力**——选择器分辨率无关＝跨品牌可行，
  且 u2 交互比 ADB-XML 坐标点击快 5 倍+（~9s vs 50s+/测试）；Media_automation（ADB-XML）作
  连不上 u2 时的稳定性兜底 + 车机技巧来源（media_session/驾驶模式）。
- **执行真实度 / 双入口**：真机走各框架真实 pytest，连不上台架自动回退模拟（保持原型节奏）；执行必落库。
  两个入口：按任务（覆盖最多的框架路由）、按脚本（脚本管理点执行，直接跑该脚本框架，不受路由）。
- **主题**：原版 + 柔和两档，柔和为默认深色主题。
- **跨品牌**：`am start` 拉起 + resource-id/text 定位 + 品牌差异收敛适配层；**选择器禁止依赖界面文本**
  （界面语言随账号可能中/英）。
