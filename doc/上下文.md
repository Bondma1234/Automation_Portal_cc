## 0.7 台架关键知识（2026-07-06 排障沉淀，勿重查）★
- **⚠ SELinux 包策略拦截自动化端口**：此固件 mangle 表 `selinux_test_automation` 链给
  6790/7912/9008 的回环流量打 `test_automation_packet_t` 标签，**非 root 域连不上 u2 服务**
  （表现：server 反复 not ready / 端口秒拒但 ss 显示 LISTEN）。
  **解法：`adb root`**（userdebug 可用；root 后 su 域宽容放行）。台架**重启后 adbd 掉回非 root**，
  平台已在 `_probe_device` 和 `core/device.py` 自动补 root。
- **残留 u2 进程会死锁霸占 9008**：异常退出的会话留死锁 app_process → 后续全部 server not ready。
  平台 `_probe_device` 与框架 `core/device.py` 已在连接前自动 `pkill -f com.wetest.uia2.Main`。
- **⚠ adb shell 管道命令必须整体作为一个参数**（2026-07-14）：`["shell","sh","-c","a | grep b"]`
  多参数经 adb 拼接会丢引号分组 → 设备上跑成全量 `dumpsys`（超时/抓错内容）。
  正确写法：`["shell", "dumpsys window | grep xxx"]`（单参数整体转发）。
- **媒体 App 会卡 FullBlockingActivity 阻塞页**（终止/异常中断可能留下）：包名在前台但主界面不可用
  → 前台判定要匹配 **Activity 类名**（KuwoMainActivity）而非包名；force-stop 冷启动可自愈
  （预检已内置该自愈重试）。
- **酷我会卡死在「正在加载…」坏状态（2026-07-13 排障）**：首页 tab 都在但 recyclerView 一直不出现、
  initScreenProgressBar 转圈不止 → 一切依赖首页加载的用例全挂。`am start -a LAUNCH_INTENT -n …`
  对已运行实例只递 intent **不会重启**（Warning: delivered to top-most instance），坏状态自我延续；
  **解法：`am force-stop com.jidouauto.media` 后冷启动即恢复**（实测 15s 内 recyclerView 出现）。
  诱因之一是并发测试会话互相打架（back/launch/uiautomator dump 抢占，dump 会拿到空 XML）。
- **.32 界面语言已变回中文**（2026-07-13 实测：我的/热门/榜单…，persist.sys.locale=zh-CN）——
  07-07 记录的 en-GB 状态已不复现；语言随账号漂移，文本选择器的风险仍在，规矩不变。
- **台架系统状态会劣化**（媒体中心黑屏 / u2 服务起不来连日志都没有）：重启台架恢复；
  重启后 adbd 掉回非 root + 可能新建驾驶员用户，引擎已自动处理 root/清残留。
- **⚠ 许可会话不稳定**：`.32` 2026-07-06 上午激活（账号 zhanghao006）→ 13:12 三条用例 PASSED
  → 16:44 起再次报 license not valid（外网正常）。疑似同账号在别处登录被挤掉线
  （比如拿同一账号去修 `.89`）。**跑真机回归前先确认台架上酷我能手动打开**。
- **台架现状（2026-07-13 实测更新）**：`.32`=主力（中文界面已恢复；酷我偶发卡「正在加载」坏状态，
  force-stop 冷启动可自愈）；`.89`=**已恢复可用**——adb root 通、u2 通、zh-CN、media 版本与 .32
  一致（CL54.26.225）、酷我冷启动 15s 内首页全量加载（许可问题消失）。两台都是 AUDI MMI 1920×816。
- **酷我选择器增量**（.32 复验，POM 已按此更新）：
  播放条 `id_op_prev/playpause/next`、曲名 `tv_title` 与 P0 一致；
  **搜索结果项 = `audio_name`（不是 P0 记的 `title`）**；搜索输入框 = `editText`；
  主页默认可能落「My」卡片页（无歌曲列表）→ 播放入口统一走 搜索→第一条（ensure_playing）；
  界面语言随账号可能中/英 → 选择器禁止依赖界面文本；
  主页标志 = 工具栏「###搜索」按钮（recyclerView 不可靠：搜索页也有）。

## 7. 关键架构决策记录（重建版）
- **前端不用 Vue/Element Plus 工程化**：硬性要求是与原型完全一致，原型是自定义 token 的原生实现；上组件库必偏离观感，上打包链对 Python 团队维护成本高。用 ES Modules 分层（api/store/ui/theme/router/views 每模块一文件）达成分层诉求。若后期要 Vue 化，api.js/store.js 分层可平移。
- **执行引擎策略式**：SimulatedRunner（默认，节奏与原型一致）/ 真机 pytest（替换 `_execute_cell` 一个函数）。job 内存态 + 完成即落库。
- **覆盖率分子单一收口**：一切来源（种子/上传/仓库扫描）都写 `script_case`，页面显示与看板计算天然一致。
- **种子数据与原型对齐**：详见 §5；`backend/app/db.py` 的 SEED_* 常量即原型内置数据。
