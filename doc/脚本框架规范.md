# JDO 自动化脚本框架规范（脚本作者必读）

> 版本：v0.1 · 2026-06-25 · 适用：`framework/` 下编写用例脚本的同学  
> 平台只认**符合本规范**的脚本；不符合规范的脚本无法被正确执行/统计覆盖率。

---

## 0. 角色分工
- **平台组（我们）维护**：`framework/core/`（设备/adb/基类）、`framework/conftest.py`、`platform/`（平台与执行引擎）、本规范。**不要改这些**，有需求提给平台组。
- **脚本作者（你）编写**：`framework/pages/<模块>/`（页面对象）、`framework/testcases/<模块>/`（用例）。

---

## 1. 目录与命名约定（务必遵守）
```
framework/
├─ pages/<模块>/xxx_page.py     # 页面对象（Page Object）
├─ testcases/<模块>/test_*.py   # 用例，文件必须 test_ 开头
```
- 模块目录名用英文小写：`kuwo` / `ximalaya` / `iqiyi` / `leting` …
- 用例文件：`test_<功能>.py`；用例函数：`test_<场景>()`。
- 每个目录放 `__init__.py`（空文件即可）。

---

## 2. 三条强制规则（违反则平台不收）

### 规则①：用例必须标 `@pytest.mark.case("用例编号")`
平台靠它统计覆盖率、建脚本↔手工用例映射。**没标 = 不计入覆盖率**。
```python
@pytest.mark.case("KW-PLAY-005")
def test_kuwo_switch_next(kuwo):
    ...
```

### 规则②：禁止在用例里写裸选择器，必须走 Page Object
用例只描述业务步骤；元素定位封装在 `pages/`。这样换 driver / 改 UI 只改一处。
```python
# ❌ 禁止
kuwo(resourceId="com.jidouauto.media:id/id_op_next").click()
# ✅ 正确
KuwoPlayerPage(kuwo).next()
```

### 规则③：每条用例开头先“归位”，不要依赖上一条的残留状态
用例之间互相独立。酷我用 `ensure_home()`（基类已提供）回到主页再操作。

---

## 3. 关键约定

- **设备句柄**：用例参数注入 `kuwo`（=当前设备，conftest 提供）；不要自己 `u2.connect`。
- **拉起 App / 子模块**：用 `am start`（封装在 `core.adb_helper.am_start`），不点 Launcher（品牌无关）。
  Oneinfo 是**一个包 `com.jidouauto.media` 内含 4 模块**，各模块独立 Activity（见 `config/settings.py` 的 `ACTIVITIES`）。
- **选择器优先级**：`resourceId` > `text` > `descriptionContains`。
  注意本 App `content-desc` 是非标值（如 `###搜索`），匹配用 `descriptionContains` 而非精确 `description`。
- **媒体类校验“真的在播放”**：用 `page.media_state()`（底层 `dumpsys media_session`）判断 `state==PLAYING` + 曲目，**不要只看 UI**。
- **断言**：用明确的中文断言信息，便于失败定位。
- **失败现场**：conftest 已自动采集截图+UI层级（接 Allure 时附到报告），用例里无需手动截图。

---

## 4. 写一条用例的标准步骤
1. 在 `pages/<模块>/` 加/改页面对象，封装这条用例要用的元素与动作（继承模块基类）。
2. 在 `testcases/<模块>/test_xxx.py` 写用例，标 `@pytest.mark.case`。
3. 本地真机自测：`cd framework && python -m pytest testcases/<模块>/test_xxx.py -v`（设备用环境变量 `JDO_DEVICE` 指定）。
4. 通过后提交；平台会扫描标记、纳入覆盖率，并可在执行中心调度。

## 5. 参考样例（酷我，已通过真机验证）
- 页面对象：`pages/kuwo/base.py`（`ensure_home`）、`kuwo_player_page.py`（播放/暂停/切歌）、`kuwo_search_page.py`（搜索）。
- 用例：`testcases/kuwo/test_kuwo_play.py`、`test_kuwo_switch.py`、`test_kuwo_search.py`。
- **新模块（喜马拉雅/爱奇艺/乐听）照此结构复制扩展即可。**

---

## 6. 平台如何消费你的脚本
- **覆盖率**：`GET /api/coverage` 扫描所有 `@pytest.mark.case` → 已自动化用例集。
- **执行**：`POST /api/run {target, device}` → 执行引擎 subprocess 调 pytest（带 json 报告 + Allure），解析出每条用例 `outcome + case_id` 返回。
- 所以：**只要遵循本规范，脚本写完即可被平台执行与统计，无需平台改代码。**
